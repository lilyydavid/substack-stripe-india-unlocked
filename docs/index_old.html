Unblocking Indian Creator Payouts on Substack

Indian creators with audiences of 10,000–50,000 subscribers are generating close to $0 in Substack GMV. Not because the audience isn't there — because the payment infrastructure is frozen.

Why now
Indian creators aren't waiting. They're wiring Razorpay, Instamojo, and Patreon into their Substack pages and building paid operations permanently off-platform. Every month this persists, the migration back gets harder — and Substack's position as the default paid newsletter platform in India weakens.

Stripe India moved to an invite-only merchant model in response to RBI regulatory tightening — mandatory FIRA tracking, a 6-month forex realisation cap, and EDPMS compliance requirements that make per-creator manual review uneconomical. Substack, which routes all transactions exclusively through Stripe, had no fallback. Paid subscriptions froze.

The Fix

The most tractable near-term unlock is not new payment infrastructure. It is giving Stripe's compliance team a structured, machine-readable endorsement signal from Substack that makes per-creator manual review unnecessary.

Issue Owner What it does
#101 — KYC Intake Substack Creator form capturing PAN, IFSC, bank account, RBI purpose code
#102 — Telemetry Signer Substack Signed bundle attesting creator legitimacy (account age, subscriber count, open rate, zero chargebacks)
#103 — Endorsement Bridge Stripe One optional field on POST /v1/accounts that accepts the signed bundle → 48h fast-track review vs. current 2–6 week manual queue

Issues #101 and #102 are entirely within Substack's control and have standalone value as internal compliance records. Issue #103 requires Stripe to implement one new optional field — the narrowest possible API change.

Open Questions

For Stripe India: Does your RBI compliance require the raw PAN, or will you accept a platform-attested SHA-256 hash? The answer determines the DPDPA liability surface for Substack.

For Stripe Connect: Is there an existing partner programme tier that already provides expedited review for high-trust platforms? If so, #103 may not need a new API primitive — Substack sends the bundle via a support channel instead.

For Substack: Does a platform agreement with capped joint liability for endorsed creators fit within your current legal framework?

This proposal covers KYC intake, telemetry signing, and the Stripe endorsement bridge. Escrow infrastructure, dashboard gamification, and FIRA/FIRC compliance automation are out of scope. A follow-on proposal on the escrow layer opens if the endorsement path is confirmed viable.
